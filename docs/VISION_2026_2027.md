#### 1. Identidad Visual Completa

**Problemas actuales:**
- Información densa compite por atención
- Sin respiración visual entre secciones
- Color empático pero no canonizado
- Falta jerarquía tipográfica clara

**Soluciones v2.0:**

##### A) Sistema de Color Tripartito
```
❤️ Rosa/Magenta  → Empatía (historias, comunidad)
⚙️ Verde/Cyan    → Técnica (código, arquitectura)
📚 Naranja       → Institucional (docs, guías)
```

##### B) Jerarquía Tipográfica
```
# Títulos       = Verdad (qué es)
## Subtítulos   = Contexto (por qué importa)
Párrafos        = Propósito (cómo ayuda)
> Callouts      = Acción (qué hacer)
```

##### C) Ritmo Visual
- Secciones cortas con micro-títulos
- Iconografía consistente
- Espacios de respiración cognitiva
- Secuencia narrativa clara

##### D) Posters Narrativos

- ✅ "Qué dolor repara"
- ✅ "Casos reales — 30s"
- ✅ "Filosofía del sistema"

#### 2. Desempeño y Robustez

##### A) Subtítulos en Tiempo Real (Empatía Tecnológica)

**Mejoras propuestas:**
- ✅ Conversión incremental (streaming)
- 🔄 Modelos locales cuando no hay red
- 🔄 Normalización de ruido adaptativa
- 🔄 Detección de ritmo de habla
- 🔄 Autocorrección predictiva contextual
- 🔄 Color semántico (pregunta/afirmación/emoción)
- 🔄 Modo conversación múltiple (dos voces)

**Filosofía:**
> "El humano no quiere precisión técnica.  
> El humano quiere pertenecer a la conversación."

Cada feature no es código — es **empatía tecnológica**.

##### B) Diagnóstico Electrónico (Razonamiento Causal)

**Mejoras propuestas:**
- 🔄 Pipeline de análisis formal: foto → rails → causa → reporte
- 🔄 Modos de inferencia: técnico / ingeniero / forense / educativo
- ✅ Export JSON/MD para ticketing
- 🔄 Aprendizaje incremental por técnico
- 🔄 Comparación de placas / golden reference
- 🔄 Atajos de taller (workflow optimizado)

**Filosofía:**
> "Un técnico no quiere 'poner datos'.  
> Un técnico quiere 'resolver evidencia'."

Esa diferencia separa software de herramienta.

#### 4. Ecosistema de Comunicación

##### Pitch Variants (Audiencias Múltiples)

**Medical Pitch** — Para profesionales de salud
- Enfoque: Accesibilidad auditiva en consultas
- Lenguaje: Clínico, evidence-based
- Métrica: Calidad de comunicación médico-paciente

**Repair Pitch** — Para técnicos electrónicos
- Enfoque: Diagnóstico causal en 60 segundos
- Lenguaje: Práctico, rail-first, topológico
- Métrica: Tiempo de diagnóstico reducido

**Accessibility Pitch** — Para comunidades de accesibilidad
- Enfoque: Conversaciones cara a cara sin fricción
- Lenguaje: Empático, inclusivo, ético
- Métrica: Personas conectadas sin repetir

**Education Pitch** — Para instituciones educativas
- Enfoque: Apoyo en aulas y tutorías
- Lenguaje: Pedagógico, inclusivo
- Métrica: Estudiantes que no se quedan atrás

**Open Source Pitch** — Para desarrolladores
- Enfoque: Arquitectura, APIs, contribución
- Lenguaje: Técnico, modular, bien documentado
- Métrica: Contributors, forks, real-world usage

**Ethical Tech Pitch** — Para inversionistas éticos
- Enfoque: Privacidad por diseño, cero vigilancia
- Lenguaje: Filosófico, anti-Silicon Valley
- Métrica: Impacto humano sin extractivismo

## 🏙️ Más Allá — v3.0 (2027)

### 🌆 Crear la Ciudad

**Visión:**
Iyari-ear no es solo una herramienta.  
Es el primer ladrillo de un **ecosistema de empatía técnica**.

#### Posibles Direcciones

##### 1. Iyari-suite
- `iyari-ear` — Subtítulos + Diagnóstico
- `iyari-eye` — Apoyo visual (amplificación, contraste)
- `iyari-hand` — Interfaces adaptativas (motor, cognitivo)

##### 2. Iyari-protocol
- Protocolo abierto para herramientas de apoyo
- Interoperabilidad entre apps de accesibilidad
- Estándar de privacidad por diseño

##### 3. Iyari-edu
- Plataforma educativa sobre accesibilidad técnica
- Cursos para desarrolladores sobre empatía en código
- Certificación "Ethical Tech Engineer"

##### 4. Iyari-mesh
- Red descentralizada de herramientas de apoyo
- Sin servidores centrales
- Peer-to-peer, local-first

**Nota importante:**
> Estas son **posibilidades**, no promesas.  
> La dirección la marca la comunidad y las necesidades reales.

## 📊 Métricas que Sí Importan

### Métricas Humanas (Las Únicas)

**1. Historias Reales**
- Cuántas personas comparten testimonios
- Diversidad de casos de uso
- Impacto emocional reportado

**2. Contribuciones Significativas**
- PRs que mejoran accesibilidad
- Traducciones a más idiomas
- Documentación mejorada
- Bugs reportados con contexto

**3. Distribución Ética**
- Descargas sin tracking
- Instalaciones en comunidades marginadas
- Uso en contextos educativos
- Adopción en talleres de reparación

### Métricas que NO Medimos

- ❌ Daily Active Users (DAU)
- ❌ Engagement rate
- ❌ Time on app
- ❌ Conversion funnels
- ❌ Retention curves
- ❌ Growth hacking metrics

**Razón:**
> "Si el objetivo es ayudar, la métrica es simple:  
> ¿Ayudó? Sí o no."
